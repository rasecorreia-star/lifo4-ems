Capitulo 1
Introducao ao Sistema

1.1 O que e o Lifo4 EMS

O Lifo4 EMS (Energy Management System) e uma plataforma completa e moderna para gerenciamento de sistemas de armazenamento de energia em baterias (BESS - Battery Energy Storage System). Desenvolvido com tecnologias de ponta, o sistema oferece monitoramento em tempo real, controle avancado, e inteligencia artificial para otimizacao de operacoes.

Principais Beneficios

  • Monitoramento 24/7: Acompanhe todos os parametros do seu sistema em tempo real
  • Otimizacao com IA: Algoritmos inteligentes para maximizar eficiencia e lucro
  • Seguranca Avancada: Deteccao de anomalias e prevencao de thermal runaway
  • Integracao Total: Conecte-se a rede eletrica, mercados de energia e outros sistemas

Funcionalidades Principais

O Dashboard Principal e a tela inicial do sistema, oferecendo uma visao consolidada de todos os sistemas BESS sob seu gerenciamento. Aqui voce encontra metricas essenciais, alertas ativos e indicadores de performance em tempo real.

Elementos do Dashboard

2.3 Graficos e Indicadores

Grafico de Potencia

O grafico principal mostra a potencia ao longo do tempo. A area verde indica descarga (fornecendo energia) e a area azul indica carga (consumindo energia). Voce pode ajustar o periodo visualizado usando os botoes de intervalo (1h, 6h, 24h, 7d, 30d).

Indicador de Receita

Mostra a receita/economia gerada no periodo selecionado, calculada com base nas tarifas de energia e nos ciclos de carga/descarga realizados.

Dica: Personalizacao

Voce pode personalizar os cards exibidos no dashboard atraves do menu de Configuracoes > Preferencias > Dashboard. Arraste e solte os widgets para reorganizar o layout conforme sua preferencia.

3.3 Painel de Controle

O Painel de Controle (/systems/:id/control) permite operar o sistema BESS manualmente ou configurar modos automaticos de operacao.

Atencao

Operacoes manuais no Painel de Controle requerem permissoes de nivel Operador ou superior. Certifique-se de entender as implicacoes antes de alterar parametros do sistema.

Modos de Operacao

  • Automatico: O sistema opera baseado em regras pre-configuradas e otimizacao por IA
  • Manual: Controle total sobre carga/descarga
  • Peak Shaving: Reduz picos de demanda automaticamente
  • Arbitragem: Compra energia barata e vende cara
  • Backup: Mantem reserva para emergencias
  • Grid Services: Participa de servicos ancilares

Comandos Disponiveis

  • Iniciar Carga: Inicia carregamento com potencia especificada
  • Iniciar Descarga: Inicia descarga com potencia especificada
  • Parar: Interrompe operacao atual (standby)
  • Definir SOC Alvo: Sistema carrega/descarrega ate atingir SOC desejado
  • Emergencia: Desliga sistema imediatamente (use apenas em emergencias)

Capitulo 4
Monitoramento de Cameras

4.1 Lista de Cameras

O modulo de Cameras (/cameras) permite gerenciar todas as cameras de seguranca e monitoramento termico instaladas nos sites BESS. O sistema suporta cameras IP padrao, cameras PTZ (Pan-Tilt-Zoom), cameras dome e cameras termicas.

Tipos de Cameras Suportadas

Tipo Descricao Uso Recomendado
Padrao Cameras IP fixas de vigilancia Monitoramento geral, perimetro
PTZ Cameras com controle de movimento Areas amplas, rastreamento
Dome Cameras em domo, discretas Areas internas, salas tecnicas
Termica Cameras com sensor infravermelho Monitoramento de temperatura BESS

Funcionalidades da Lista

  • Visualizacao em Grid: Cards com preview de cada camera
  • Visualizacao em Lista: Formato tabular com mais detalhes
  • Multiview: Visualize multiplas cameras simultaneamente
  • Filtros: Por status (online/offline), tipo, localizacao
  • Busca: Encontre cameras por nome ou local

Gerenciamento de Cameras

Para cada camera, voce pode:

  • Adicionar: Clique em "Nova Camera" e preencha os dados (IP, modelo, localizacao)
  • Editar: Clique nos 3 pontinhos > Editar para modificar informacoes
  • Apagar: Clique nos 3 pontinhos > Apagar para remover (requer confirmacao)
  • Ver ao Vivo: Clique no card para abrir visualizacao em tempo real

4.2 Cameras Termicas

As cameras termicas sao essenciais para a seguranca de sistemas BESS, permitindo detectar elevacoes de temperatura que podem indicar falhas ou risco de thermal runaway.

Deteccao de Thermal Runaway

O Lifo4 EMS monitora continuamente as imagens termicas e alerta automaticamente quando detecta temperaturas acima dos limites configurados. Isso permite acao rapida antes que problemas se tornem criticos.

Pagina de Monitoramento Termico

Acesse /cameras/thermal para uma visao dedicada das cameras termicas com:

  • Leituras em Tempo Real: Temperatura minima, maxima e media de cada camera
  • Historico de Temperatura: Grafico mostrando variacao nas ultimas horas
  • Zonas Termicas: Defina areas especificas para monitoramento (racks, inversores, etc.)
  • Alertas: Lista de alertas termicos com opcao de reconhecimento

Limites de Temperatura

Nivel Temperatura Acao
Normal < 35C Operacao normal
Atencao 35C - 45C Monitoramento intensificado
Alerta 45C - 55C Notificacao para operadores
Critico > 55C Alarme + Acao automatica

4.3 Multiview

O modo Multiview permite visualizar varias cameras simultaneamente em uma unica tela. E ideal para salas de controle e monitoramento centralizado.

Layouts Disponiveis

  • 2x2: 4 cameras em grade
  • 3x3: 9 cameras em grade
  • 4x4: 16 cameras em grade

Dica: Tela Cheia

Para melhor experiencia no Multiview, pressione F11 para entrar em modo tela cheia no navegador. As cameras serao redimensionadas automaticamente.

Capitulo 5
Microgrids

5.1 Dashboard de Microgrids

O modulo de Microgrids (/microgrids) oferece gerenciamento completo de microrredes, permitindo operar sistemas que combinam geracao renovavel (solar, eolica), armazenamento em baterias e conexao com a rede eletrica principal.

[Captura de Tela: Dashboard de Microgrids]
Mostrando fluxo de potencia, modo de operacao e componentes
Figura 5.1 - Dashboard de Microgrids

Componentes de um Microgrid

  • Geracao Solar: Paineis fotovoltaicos e inversores
  • Geracao Eolica: Turbinas eolicas (quando aplicavel)
  • Armazenamento BESS: Sistemas de baterias integrados
  • Gerador Backup: Geradores diesel ou gas natural
  • Conexao Grid: Ponto de conexao com a rede eletrica
  • Cargas: Consumidores conectados ao microgrid

5.2 Modos de Operacao

O microgrid pode operar em diferentes modos, cada um adequado para diferentes situacoes:

Modo Descricao Quando Usar
Conectado a Rede Microgrid sincronizado com a rede eletrica principal Operacao normal, otimizacao economica
Ilhado Microgrid operando independentemente da rede Queda de energia, manutencao da rede
Black Start Processo de religamento apos apagao Recuperacao pos-blackout
Transicao Mudando entre modos de operacao Automatico durante mudancas

Inversor Grid-Forming (GFM)

O sistema indica quando o inversor esta operando em modo Grid-Forming, essencial para operacao ilhada. Neste modo, o inversor e responsavel por estabelecer a referencia de tensao e frequencia para todo o microgrid.

Monitoramento de Frequencia

O dashboard exibe a frequencia do microgrid em tempo real com indicacao visual:

  • 60.00 Hz: Frequencia nominal (verde)
  • 59.95 - 60.05 Hz: Faixa normal de operacao (verde)
  • Fora da faixa: Indica necessidade de ajuste (amarelo/vermelho)

5.3 Grid Services

O microgrid pode participar de servicos ancilares da rede, gerando receita adicional:

FCR (Frequency Containment Reserve)

Reserva rapida para estabilizacao de frequencia. O sistema responde automaticamente a desvios de frequencia em milissegundos.

aFRR (Automatic Frequency Restoration Reserve)

Reserva automatica para restauracao de frequencia. Ativada por sinal do operador do sistema apos evento de frequencia.

Controle de Tensao

Fornecimento de potencia reativa para suporte de tensao na rede local.

Peak Shaving

Reducao de picos de demanda para diminuir custos com demanda contratada.

Aba de Previsao

A aba "Previsao" no dashboard mostra a previsao de geracao renovavel baseada em dados meteorologicos:

  • Previsao de irradiacao solar para proximas 24-48 horas
  • Previsao de velocidade do vento (se aplicavel)
  • Estimativa de geracao em kWh
  • Recomendacoes de operacao baseadas na previsao
Capitulo 6
Analytics e Otimizacao

6.1 Analytics

O modulo de Analytics (/analytics) fornece analises detalhadas sobre a performance dos sistemas BESS, incluindo metricas de eficiencia, ciclos, receita e tendencias.

Metricas Principais

  • Eficiencia Round-Trip: Razao entre energia descarregada e carregada
  • Fator de Capacidade: Utilizacao da capacidade instalada
  • Ciclos Realizados: Numero de ciclos completos no periodo
  • Energia Processada: Total de energia carregada/descarregada
  • Receita Gerada: Ganhos com arbitragem e servicos
  • Disponibilidade: Percentual de tempo operacional

Relatorios Disponives

Relatorio Conteudo Periodicidade
Performance Diaria Resumo de operacoes do dia Diario
Analise Semanal Tendencias e comparativos Semanal
Relatorio Mensal KPIs, financeiro, manutencao Mensal
Benchmarking Comparacao entre sistemas Sob demanda

6.2 Saude da Bateria

A pagina de Saude da Bateria (/battery-health) monitora a degradacao e fornece estimativas de vida util restante dos sistemas BESS.

Indicadores Monitorados

  • SOH (State of Health): Capacidade atual vs. capacidade nominal
  • Resistencia Interna: Aumento indica degradacao
  • Capacidade Efetiva: kWh reais disponiveis
  • Ciclos Totais: Historico completo de ciclos
  • Vida Util Estimada: Anos restantes de operacao
Modelo PyBAMM

O Lifo4 EMS utiliza o framework PyBAMM para modelagem eletroquimica avancada das baterias, permitindo previsoes precisas de degradacao e otimizacao de ciclos para maximizar a vida util.

6.3 Otimizacao de Energia

O modulo de Otimizacao (/optimization) utiliza algoritmos de IA para determinar a melhor estrategia de operacao considerando precos de energia, previsao de demanda e restricoes do sistema.

Fatores Considerados

  • Tarifas de energia (hora-hora)
  • Previsao de geracao solar/eolica
  • Previsao de demanda de carga
  • Custos de degradacao da bateria
  • Compromissos de servicos ancilares
  • Restricoes operacionais (SOC, temperatura)

Resultados da Otimizacao

O sistema gera um plano de operacao para as proximas 24-48 horas incluindo:

  • Perfil de carga/descarga otimo
  • Receita esperada
  • Consumo de ciclos estimado
  • Comparativo com estrategia atual
Capitulo 7
Trading e Mercado de Energia

7.1 Energy Trading

O modulo de Energy Trading (/trading) permite participar de mercados de energia, realizando compra e venda de energia atraves dos sistemas BESS.

Mercados Suportados

  • Mercado Spot: Negociacao para entrega imediata
  • Day-Ahead: Leiloes para dia seguinte
  • Intraday: Ajustes durante o dia
  • Servicos Ancilares: Reservas e regulacao
  • Bilateral: Contratos diretos com consumidores

Dashboard de Trading

A interface apresenta:

  • Precos atuais e historico
  • Posicoes abertas e fechadas
  • P&L (Profit and Loss) em tempo real
  • Grafico de precos com indicadores
  • Livro de ofertas

7.2 Trading com IA

O Trading Dashboard com IA (/trading-dashboard) utiliza algoritmos de Deep Reinforcement Learning para executar estrategias automaticas de trading.

Deep Reinforcement Learning

O modelo de IA foi treinado com anos de dados historicos de precos e aprende continuamente com cada operacao. Ele considera multiplos fatores como precos, volatilidade, horarios de pico e correlacoes para maximizar retornos.

Modos de Operacao

  • Automatico: IA executa trades autonomamente
  • Semi-automatico: IA sugere, operador aprova
  • Manual: Apenas previsoes e analises

Gerenciamento de Risco

  • Limites de exposicao maxima
  • Stop-loss automatico
  • Diversificacao de estrategias
  • Alertas de volatilidade

7.3 Demand Response

O modulo de Demand Response (/demand-response) gerencia a participacao em programas de resposta a demanda, onde o sistema reduz ou desloca carga em resposta a sinais do operador da rede.

Tipos de Eventos

Tipo Notificacao Duracao Tipica
Economico 24h antecedencia 4-8 horas
Emergencia 30min - 2h 1-4 horas
Reserva 10-30 min 15min - 2h
Regulacao Segundos Continuo
Capitulo 8
Digital Twin e Simulacao

8.1 Digital Twin

O Digital Twin (/digital-twin) e uma replica virtual do sistema BESS fisico, sincronizada em tempo real e capaz de simular comportamentos futuros.

[Captura de Tela: Digital Twin]
Visualizacao 3D do sistema com dados em tempo real
Figura 8.1 - Interface do Digital Twin

Funcionalidades

  • Visualizacao 3D: Representacao visual do sistema fisico
  • Dados em Tempo Real: Parametros sincronizados com equipamento real
  • Simulacao What-If: Teste cenarios sem afetar sistema real
  • Previsao: Estimativas de comportamento futuro
  • Diagnostico: Identificacao de anomalias por comparacao

Modelo PyBAMM Integrado

O Digital Twin utiliza modelos eletroquimicos do PyBAMM para simular o comportamento interno das celulas de bateria, incluindo:

  • Distribuicao de corrente nas celulas
  • Gradiantes de temperatura internos
  • Envelhecimento e degradacao
  • Comportamento em condicoes extremas

8.2 Simulacao

O modulo de Simulacao (/simulation) permite testar estrategias de operacao e avaliar performance em diferentes cenarios.

Cenarios Pre-definidos

  • Arbitragem Pura: Compra baixo, vende alto
  • Peak Shaving: Reducao de picos de demanda
  • Backup de Emergencia: Reserva para falhas de rede
  • Integracao Solar: Otimizacao com geracao PV
  • Servicos Ancilares: Participacao em regulacao

Parametros de Simulacao

  • Periodo de simulacao (dias, meses, anos)
  • Perfil de precos de energia
  • Perfil de carga do consumidor
  • Geracao renovavel disponivel
  • Parametros da bateria
  • Custos operacionais

Resultados

Ao final da simulacao, o sistema apresenta:

  • Receita/economia total
  • ROI (Return on Investment)
  • Payback estimado
  • Degradacao da bateria
  • Graficos de operacao
  • Comparativo entre estrategias
Capitulo 9
Manutencao e Alertas

9.1 Sistema de Alertas

O modulo de Alertas (/alerts) centraliza todas as notificacoes do sistema, categorizadas por severidade e tipo.

Niveis de Severidade

Nivel Cor Exemplos Acao
Info Azul Inicio de carga, mudanca de modo Informativo
Aviso Amarelo SOC baixo, temperatura elevada Monitorar
Alerta Laranja Limite ultrapassado, falha comunicacao Investigar
Critico Vermelho Thermal runaway, falha sistema Acao imediata

Configuracao de Alertas

Na pagina de Configuracao de Alarmes (/alarm-config), voce pode personalizar os limites e acoes para cada tipo de alerta:

  • Definir thresholds personalizados
  • Configurar destinatarios de email/SMS
  • Definir acoes automaticas
  • Agendar periodos de silencio

9.2 Manutencao Preventiva

O modulo de Manutencao (/maintenance) gerencia o calendario de manutencoes preventivas e historico de intervencoes.

Tipos de Manutencao

  • Inspecao Visual: Verificacao de conexoes, corrosao, danos
  • Limpeza: Filtros, ventilacao, paineis
  • Testes: Isolamento, comunicacao, protecoes
  • Calibracao: Sensores de temperatura, corrente, tensao
  • Atualizacao: Firmware do BMS e outros componentes

Ordens de Servico

Na pagina de Ordens de Servico (/work-orders), voce pode:

  • Criar novas ordens de servico
  • Atribuir tecnicos responsaveis
  • Acompanhar status de execucao
  • Registrar pecas e materiais utilizados
  • Documentar com fotos e relatorios

9.3 Manutencao Preditiva com IA

O modulo de Manutencao Preditiva (/predictive) utiliza Machine Learning para prever falhas antes que ocorram.

Algoritmos Utilizados

O sistema analisa padroes em dados historicos de temperatura, tensao, corrente e outros parametros para identificar sinais de degradacao e prever necessidade de manutencao com antecedencia de dias ou semanas.

Indicadores Preditivos

  • Probabilidade de Falha: % de chance de falha nos proximos 30 dias
  • Componentes em Risco: Lista de itens com degradacao detectada
  • Recomendacoes: Acoes sugeridas para prevenir falhas
  • Impacto Estimado: Custo evitado com manutencao preventiva
Capitulo 10
Integracao e APIs

10.1 Integracao com Rede

O modulo de Integracao com Rede (/grid) gerencia a conexao do sistema BESS com a rede eletrica e operadores do sistema.

Protocolos Suportados

  • Modbus TCP/RTU: Comunicacao com inversores e medidores
  • DNP3: Protocolo para automacao de subestacoes
  • IEC 61850: Padrao para automacao de sistemas de potencia
  • OCPP: Integracao com carregadores de veiculos eletricos
  • SunSpec: Padrao para equipamentos solares

10.2 VPP (Virtual Power Plant)

O modulo VPP (/vpp) permite agregar multiplos sistemas BESS e recursos distribuidos em uma unica usina virtual.

Funcionalidades VPP

  • Agregacao de capacidade de multiplos sites
  • Despacho otimizado entre recursos
  • Participacao em leiloes de energia como entidade unica
  • Balanceamento automatico entre sistemas
  • Relatorios consolidados

10.3 API Keys

O modulo de API Keys (/api-keys) gerencia chaves de acesso para integracao com sistemas externos.

Gerenciamento de APIs

  • Criar novas chaves de API
  • Definir permissoes por chave
  • Monitorar uso e requisicoes
  • Revogar chaves comprometidas
  • Limitar taxa de requisicoes (rate limiting)
Seguranca

Mantenha suas API Keys em seguranca. Nunca compartilhe chaves em codigo publico ou repositorios. Use variaveis de ambiente para armazenar credenciais.

11.2 Configuracoes

A pagina de Configuracoes (/settings) permite personalizar diversos aspectos do sistema.

Opcoes de Configuracao

  • Perfil: Nome, email, foto, preferencias pessoais
  • Notificacoes: Email, SMS, push, frequencia
  • Interface: Tema (claro/escuro), idioma, unidades
  • Seguranca: Senha, autenticacao 2FA, sessoes ativas
  • Integracacoes: Conexoes com sistemas externos

11.3 Backup e Seguranca

O modulo de Backup (/backup) gerencia copias de seguranca dos dados e configuracoes do sistema.

Politicas de Backup

  • Automatico Diario: Backup incremental todas as noites
  • Semanal: Backup completo aos domingos
  • Mensal: Backup arquivado no primeiro dia do mes
  • Manual: Backup sob demanda a qualquer momento

Seguranca Pos-Quantica

O Lifo4 EMS implementa criptografia pos-quantica para proteger dados contra futuros computadores quanticos. Algoritmos utilizados:

  • CRYSTALS-Kyber para troca de chaves
  • CRYSTALS-Dilithium para assinaturas digitais
  • AES-256-GCM para criptografia de dados
Auditoria

Todas as acoes no sistema sao registradas no Log de Auditoria (/audit), permitindo rastreabilidade completa de quem fez o que e quando.
